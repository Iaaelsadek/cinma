
# تقرير التنظيف والتوسعة والتحسين (Cleanup & Expansion Report)

## 1. تحليل الوضع الحالي (Current Status Analysis)
تم فحص النظام بعمق واكتشاف النقاط التالية:
- **مشكلة التصنيفات (Categorization Issue)**: صفحة المسرحيات (`Plays.tsx`) كانت تعتمد على "بحث نصي" (Text Search) لقائمة عناوين ثابتة، مما أدى لظهور مسرحيات مصرية في القسم الخليجي إذا تشابهت الأسماء أو الكلمات المفتاحية.
- **نظام الاستيراد (Import System)**: السكربت الحالي (`mass_content_importer.py`) كان محدوداً (50 عنصر يومياً) ولا يقوم بالتصنيف الذكي (رمضان، مسرحيات، بلد المنشأ).
- **قاعدة البيانات (Database)**: الهيكلة جيدة ولكنها تفتقر للفهارس (Indexes) اللازمة للبحث السريع في حالة التوسع لـ 50 ألف عنصر.

## 2. الإجراءات المنفذة (Executed Actions)

### أولاً: إصلاح منطق التصنيفات (Cleanup & Categorization Fix)
- **تم تعديل `src/pages/discovery/Plays.tsx`**:
  - إلغاء الاعتماد على البحث بالاسم فقط.
  - إضافة تحقق صارم من **بلد الإنتاج (Production Country)** لكل مسرحية قبل عرضها في القسم الخليجي.
  - الآن، القسم الخليجي سيعرض فقط المسرحيات التي تحمل كود الدولة (KW, SA, AE, BH, QA, OM).

### ثانياً: التوسع في المحتوى (Content Expansion - 50k Items)
- **تم تطوير أداة جديدة `backend/mega_importer.js`**:
  - هذه الأداة مصممة لجلب 50,000 عمل فني دفعة واحدة أو على مراحل.
  - **المميزات الذكية**:
    - **تصنيف تلقائي لرمضان**: يكتشف المسلسلات الرمضانية بناءً على تاريخ العرض والدولة.
    - **تصنيف تلقائي للمسرحيات**: يكتشف المسرحيات من العناوين.
    - **تحديد بلد المنشأ**: يجلب بيانات الدولة بدقة للأعمال العربية لضمان الفرز الصحيح.
    - **دعم الاستكمال (Upsert)**: يمنع تكرار البيانات، حيث يقوم بتحديث السجلات الموجودة بدلاً من إنشاء نسخ جديدة.

### ثالثاً: تحسين الأداء وقاعدة البيانات (Performance Optimization)
- **تم إنشاء ملف تهجير جديد `supabase/migrations/20260307_optimize_movies_series.sql`**:
  - إضافة فهارس `GIN` للبحث النصي السريع (Fuzzy Search) على العناوين العربية والإنجليزية.
  - تحسين سرعة الفرز حسب التاريخ والتقييم.
  - تجهيز القاعدة لاستيعاب مئات الآلاف من السجلات دون بطء.

## 3. تعليمات التشغيل (How to Run)

### لتشغيل استيراد المحتوى الضخم (على السيرفر):
تأكد من وجود `SUPABASE_SERVICE_ROLE` في ملف `.env`، ثم نفذ:
```bash
# استيراد 50 صفحة لكل قسم (تجربة)
node backend/mega_importer.js --pages=50

# استيراد ضخم (500 صفحة = 10,000 عنصر تقريباً لكل قسم)
node backend/mega_importer.js --pages=500
```

### لتطبيق تحسينات الأداء:
قم بتنفيذ ملف SQL الجديد في Supabase SQL Editor:
`supabase/migrations/20260307_optimize_movies_series.sql`
