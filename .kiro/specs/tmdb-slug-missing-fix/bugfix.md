# وثيقة متطلبات إصلاح الخلل

## المقدمة

يطرح التطبيق خطأ "Missing slug for content" عند عرض المحتوى على الصفحة الرئيسية. يحدث هذا لأن `HomeBelowFoldSections.tsx` يجلب البيانات مباشرة من TMDB API لعدة فئات محتوى (المسلسلات الكورية، المسلسلات التركية، المسلسلات الصينية، الوثائقيات، الأنمي، الكلاسيكيات، أفلام بوليوود الاحتياطية)، واستجابات TMDB API لا تتضمن حقل `slug`. عندما يحاول `MovieCard.tsx` إنشاء عناوين URL باستخدام `generateContentUrl()` أو `generateWatchUrl()`، يطرح خطأ لأن هذه الدوال تتطلب slug صالح.

السبب الجذري هو عدم تطابق تدفق البيانات: المحتوى من TMDB API يفتقر إلى slugs، لكن دوال إنشاء URL تفرض وجود slug كمتطلب. هذا يخلق تجربة مستخدم معطلة حيث يظهر المحتوى على الصفحة لكن لا يمكن التفاعل معه.

## تحليل الخلل

### السلوك الحالي (العيب)

1.1 عندما يجلب HomeBelowFoldSections المسلسلات الكورية من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.2 عندما يجلب HomeBelowFoldSections المسلسلات التركية من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.3 عندما يجلب HomeBelowFoldSections المسلسلات الصينية من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.4 عندما يجلب HomeBelowFoldSections الوثائقيات من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.5 عندما يجلب HomeBelowFoldSections الأنمي من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.6 عندما يجلب HomeBelowFoldSections الكلاسيكيات من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

1.7 عندما يجلب HomeBelowFoldSections أفلام بوليوود كاحتياطي من TMDB API فإن النظام يعرض محتوى بدون slugs مما يسبب أخطاء "Missing slug for content" عندما يتفاعل المستخدمون مع البطاقات

### السلوك المتوقع (الصحيح)

2.1 عندما يجلب HomeBelowFoldSections المسلسلات الكورية فإن النظام يجب أن يسترجع المحتوى من CockroachDB API مع slugs صالحة أو يصفي العناصر بدون slugs قبل العرض

2.2 عندما يجلب HomeBelowFoldSections المسلسلات التركية فإن النظام يجب أن يسترجع المحتوى من CockroachDB API مع slugs صالحة أو يصفي العناصر بدون slugs قبل العرض

### السلوك غير المتغير (منع الانحدار)

3.1 عندما يجلب Home.tsx بيانات الصفحة الرئيسية الحرجة من CockroachDB API فإن النظام يجب أن يستمر في عرض المحتوى مع slugs صالحة بدون أخطاء

3.2 عندما يستقبل MovieCard محتوى مع slugs صالحة فإن النظام يجب أن يستمر في إنشاء عناوين URL صحيحة وعرض بطاقات تفاعلية

3.3 عندما يستقبل generateContentUrl محتوى مع slugs صالحة فإن النظام يجب أن يستمر في إنشاء عناوين URL محتوى صحيحة بدون طرح أخطاء

3.4 عندما يستقبل generateWatchUrl محتوى مع slugs صالحة فإن النظام يجب أن يستمر في إنشاء عناوين URL مشاهدة صحيحة بدون طرح أخطاء

3.5 عندما يتم جلب المحتوى من نقاط نهاية CockroachDB API فإن النظام يجب أن يستمر في تضمين حقول slug في الاستجابات

3.6 عندما يتفاعل المستخدمون مع بطاقات المحتوى التي لها slugs صالحة فإن النظام يجب أن يستمر في التنقل إلى صفحات المحتوى الصحيحة
